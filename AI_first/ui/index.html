What This Bundle Contains

A compact, drop-in set of docs, UI views, and templates that keep AI-led delivery structured and auditable.

  • Process docs, personas, and templates under AI_first/docs/.
  • Project planning docs under AI_first/projects/.
  • Bug management data + exports under AI_first/bugmgmt/.
  • Static UI reports under AI_first/ui/.
  • Automation scripts under AI_first/scripts/.

AI-first Philosophy

This bundle keeps AI work grounded in a clear source-of-truth: PM.html for the portfolio view, stage action files for evidence, and Bug Management for audit trails. The goal is small, reviewable changes guided by personas and documented decisions, with deeper workflow details in the Process Management page.

Optional Automation (Manual Regeneration)

Scripts are only for refreshing UI exports or scaffolding; day-to-day work stays doc-driven and AI-led.

  • python3 AI_first/scripts/render_docs.py — refresh AI_first/ui/docs/ from markdown.
  • python3 AI_first/scripts/render_pm.py — refresh AI_first/ui/PM.html and project detail pages.
  • python3 AI_first/scripts/watch_docs.py — auto-render docs while you edit.
  • python3 AI_first/scripts/init_project.py — scaffold new project docs and update PM.html.
  • python3 AI_first/scripts/issues.py list — regenerate Bug Management JSON/HTML exports.

Keep AI_first/ intact; treat it as the drop-in bundle for process, Project Management, and Bug Management.
