Verification Checklist

  1. Confirm each project row reflects the current phase and status.
  2. Verify ownership and last updated dates match project docs.
  3. Review open items in BugMgmt and confirm prefixes.
  4. Check user docs status and ensure build steps are current.

Start a New Repo (AI-first)

  1. Tell the agent the new repo name and purpose; it will copy/rename the repo and treat the template as read-only.
  2. Provide the project name, prefix, and owner; the agent runs support/scripts/init_project.py and updates support/docs/projectplan.md, this PM page, and a project detail page.
  3. Ask the agent to keep rendered docs current by running python3 support/scripts/watch_docs.py from the repo root while you edit, or python3 support/scripts/render_docs.py after doc edits.
  4. Ask the agent to update BugMgmt prefixes/owners and regenerate exports after any bug changes.
  5. Product work lives in src/, tests/, config/, data/, assets/, scripts/, docs/; keep support/ for tooling.
