AI_first Drop-in Pack

AI_first is a self-contained support bundle that adds process management, project management, and bug management to any repo. Copy the AI_first/ folder into a new codebase, keep it intact, and use the docs and UI with your AI partner to run planning, execution, and QA in one place.

What This Repo Contains

  • Process docs, personas, and templates under AI_first/docs/.
  • Project planning docs under AI_first/projects/.
  • Bug management data + exports under AI_first/bugmgmt/.
  • Static UI reports under AI_first/ui/.
  • Automation scripts under AI_first/scripts/.

AI-first Workflow

  1. Copy the entire AI_first/ directory into your target repo root.
  2. Open index.html or AI_first/ui/PM.html via file:// to get oriented.
  3. Read the primary docs: AI_first/docs/process.md, AI_first/docs/projectplan.md, and AI_first/docs/project_wide_docs/personas.md.
  4. Pick the active project/phase and keep the stage action file open as the working scratchpad.
  5. Use the personas to drive decisions: Project Mgmt defines actions, Developer executes work, QA reports bugs and proposes fixes.
  6. Keep PM.html, project docs, and BugMgmt entries updated as you go.

Optional Automation (Manual Regeneration)

  • python3 AI_first/scripts/render_docs.py — refresh AI_first/ui/docs/ from markdown.
  • python3 AI_first/scripts/watch_docs.py — auto-render docs while you edit.
  • python3 AI_first/scripts/init_project.py — scaffold new project docs and update PM.html.
  • python3 AI_first/scripts/issues.py list — regenerate BugMgmt JSON/HTML exports.
